## 1. 프로젝트 개요

### 1.1 서비스 소개

| 항목 | 내용 |
|------|------|
| **타겟 고객** | 공교육(학교) 선생님 |
| **서비스 목적** | 학교 현장에서 손이 많이 가고 시간이 많이 걸리는 작업 단축 |
| **핵심 컨셉** | 쉽고, 간편한 학생 관리 시스템 + 재미있고 체계적인 수업 서비스 |

### 1.2 사용 환경

| 사용자 | 디바이스 | 주요 활동 |
|--------|----------|-----------|
| **선생님** | 데스크탑/노트북 | 관리, 대시보드, 리포트, 수업 진행 |
| **학생** | 태블릿 PC | 교과서, 문제 풀이, 필기, 제출 |

### 1.3 예시 컨텍스트

- **학교:** 비상 중학교
- **과목:** 수학1
- **학급:** 1학년 3반 (28명)
- **선생님:** 윤지명

## 2. 기존 IA 구조 분석

### 2.1 기존 구조

```
1_홈
  ├ 학급 현황
  ├ 대시보드  
  └ 리포트

2_교과서 (서브탭 없음, 바로 페이지 전환)

3_과제
  ├ 과제 목록 보기
  └ 과제 게시판

4_평가
  └ 평가 목록 보기

5_수업 자료실
  ├ 수업 자료
  └ 내 자료
```

### 2.2 문제점 분석

#### 🔴 네비게이션 일관성 부재

| 메뉴 | 서브탭 수 | 문제 |
|------|-----------|------|
| 홈 | 3개 | - |
| 교과서 | 0개 | 바로 전환 |
| 과제 | 2개 | - |
| 평가 | 1개 | 불필요한 서브탭 |
| 자료실 | 2개 | - |

- 서브탭 개수가 0~3개로 불규칙
- 클릭 결과 예측 불가능
- "교과서"만 클릭 시 바로 페이지 전환

#### 🔴 용어 혼란

| 문제 | 설명 |
|------|------|
| "홈" vs "학급 현황" | 관계가 불명확 |
| "과제 목록 보기" vs "과제 게시판" | 차이점 모호 |
| "수업 등록" / "수업 시작" / "수업 설계하기" | 버튼 역할 혼란 |

#### 🔴 교과서 배치 이슈

- 홈, 과제, 평가, 자료실 = **관리 시스템** (CRUD)
- 교과서 = **수업 진행** (응시 페이지)
- 성격이 다른 메뉴가 같은 레벨에 혼재

## 4. 화면별 기능 정의

### 4.1 홈 > 오늘

> **핵심 질문:** "오늘 내가 뭐 해야 하지?"

#### 화면 구성

| 영역 | 기능 | 우선순위 |
|------|------|----------|
| **빠른 도구** | 재구성, 의견보드, 스마트 수업도구 | ⭐⭐⭐ |
| **오늘의 할 일** | 채점 필요 N건, 마감 임박 N건, 오늘 수업 N건 | ⭐⭐⭐ |
| **이번 주 일정** | 주간 캘린더 (수업/숙제/시험 표시) | ⭐⭐⭐ |
| **알림장** | 오늘의 공지사항 작성/확인 | ⭐⭐ |
| **최근 활동** | 최근 진행한 수업/숙제/시험 (바로가기) | ⭐⭐ |
| **메모** | 개인 메모 / 학생별 메모 (탭 전환) | ⭐⭐ |

#### 빠른 도구 상세

| 도구 | 설명 | 아이콘 |
|------|------|--------|
| **재구성** | 자료를 편집하여 숙제/시험 만들기 | ✏️ |
| **의견보드** | 학생 의견 수집, 투표 | 💬 |
| **스마트 수업도구** | 타이머, 랜덤 뽑기 등 | 🎮 |

### 4.2 홈 > 우리 반

> **핵심 질문:** "우리 반 애들 상황이 어때?"

#### 화면 구성

| 영역 | 기능 | 우선순위 |
|------|------|----------|
| **빠른 도구** | 심리검사, 마이룸, 목표 설정, 상점 | ⭐⭐⭐ |
| **이번 주 제출 현황** | 제출률 %, 미제출 학생, 알림 보내기 | ⭐⭐⭐ |
| **리워드 TOP 5** | 상위 학생 랭킹, 포인트 현황 | ⭐⭐ |
| **학생 리스트** | 전체 학생 목록, 검색, 정렬, 메시지 | ⭐⭐⭐ |

#### 빠른 도구 상세 (우선순위순)

| 순위 | 도구 | 설명 | 아이콘 |
|------|------|------|--------|
| 1 | **심리검사** | 학습심리검사 진행/결과 | 🧠 |
| 2 | **마이룸** | 학생 마이룸 꾸미기 | 🏠 |
| 3 | **목표 설정** | 이번 주 학습 목표 설정 | 🎯 |
| 4 | **상점** | 리워드 포인트 사용 | 🏪 |

#### 학생 리스트 컬럼

| 컬럼 | 설명 |
|------|------|
| No | 출석번호 |
| 이름 | 학생 이름 |
| 오늘 기분 | 😊😐😢 이모지 표시 |
| 제출 현황 | ✅ 2/2, ⚠️ 1/2, ❌ 0/2 |
| 리워드 | ❤️ 포인트 |
| 메모 | 📌 메모 있음 표시 |
| 메시지 | 💬 개인 메시지 버튼 |

### 4.3 메시지 기능

#### 기능 플로우

```
우리 반 > 학생 리스트
    │
    ├─ [💬 메시지] 클릭 → 개별 채팅 페이지
    │                         │
    │                         └─ [📋 전체 보기] → 전체 메시지 페이지
    │                                                    │
    │                                                    └─ 학생 클릭 → 개별 채팅
    │
    └─ [💬 전체 메시지] 버튼 → 전체 메시지 페이지
```

#### 개별 채팅 페이지

| 요소 | 설명 |
|------|------|
| 헤더 | 학생 프로필 (이름, 학년반, 번호) |
| 채팅 영역 | 메시지 히스토리 (선생님/학생 구분) |
| 입력 영역 | 텍스트 입력 + 보내기 버튼 |
| 전체 보기 | 모든 학생 대화 목록으로 이동 |

#### 전체 메시지 페이지

| 섹션 | 설명 |
|------|------|
| 최근 대화 | 메시지 히스토리가 있는 학생 목록 |
| 새 대화 시작 | 아직 대화 없는 학생 목록 |

### 4.4 분석 vs 결과 차이

| 구분 | 분석 (대시보드) | 결과 (리포트) |
|------|-----------------|---------------|
| **데이터** | 집계/통계 (% 수치화) | 원본 그대로 (O/X) |
| **관점** | 숲을 봄 (전체 수준) | 나무를 봄 (개별 상세) |
| **범위** | 전체 → 단원 → 개념 | 활동 → 학급 → 학생 → 문항 |
| **질문** | "우리 반 수준이 어때?" | "이 평가에서 누가 뭘 틀렸어?" |

### 4.5 재구성 기능 배치

> 별도 IA 없이 버튼으로 해결

#### 진입점

| 위치 | 방식 |
|------|------|
| **홈 > 오늘 > 빠른 도구** | [✏️ 재구성] 버튼 클릭 |
| **숙제 목록 화면** | [+ 새로 만들기] 버튼 → 재구성 도구 |
| **시험 목록 화면** | [+ 새로 만들기] 버튼 → 재구성 도구 |
| **자료실 > 자료 선택 시** | [숙제로 출제] [시험으로 출제] 버튼 |

## 5. 디자인 시스템

### 5.1 디자인 컨셉: "Nano Banana"

| 항목 | 스타일 |
|------|--------|
| **전체 톤** | 깔끔하고 부드러운 UI |
| **모서리** | 둥근 모서리 (rounded-2xl, rounded-3xl) |
| **그림자** | 부드러운 그림자 (Soft shadows) |
| **색상** | 파스텔 그라데이션 |
| **인터랙션** | 생동감 있는 호버/클릭 애니메이션 |

### 5.2 폰트

```css
font-family: 'Pretendard', -apple-system, BlinkMacSystemFont, sans-serif;
```

### 5.3 그림자 스타일

```css
/* 카드 기본 */
box-shadow: 0 2px 12px rgba(0, 0, 0, 0.06);

/* 버튼 강조 */
box-shadow: 0 4px 12px rgba(59, 130, 246, 0.3);

/* LNB */
box-shadow: 4px 0 20px rgba(0, 0, 0, 0.05);
```

### 5.4 색상 팔레트

| 용도 | 색상 |
|------|------|
| Primary | `#3B82F6` (Blue-500) |
| 채점 필요 | `#EF4444` (Red) |
| 마감 임박 | `#F97316` (Orange) |
| 오늘 수업 | `#3B82F6` (Blue) |
| 완료 | `#22C55E` (Green) |

### 5.5 빠른 도구 그라데이션

| 도구 | 그라데이션 |
|------|------------|
| 재구성 | `from-yellow-50 to-yellow-100` |
| 의견보드 | `from-blue-50 to-blue-100` |
| 스마트도구 | `from-orange-50 to-orange-100` |
| 심리검사 | `from-pink-50 to-pink-100` |
| 마이룸 | `from-blue-50 to-blue-100` |
| 목표설정 | `from-green-50 to-green-100` |
| 상점 | `from-yellow-50 to-yellow-100` |

### 5.6 LNB 스타일

| 항목 | 스타일 |
|------|--------|
| 기본 너비 | 256px (w-64) |
| 접힘 너비 | 80px (w-20) |
| 아코디언 기본값 | **모두 접힘** |
| 펼침 애니메이션 | `max-h` 트랜지션 |
| 스크롤 | 없음 (overflow-hidden) |

## 6. 개선 우선순위

### Priority 1 - 즉시 개선

| 항목 | 설명 |
|------|------|
| 네비게이션 명확화 | 서브탭 정리, 용어 통일 (과제→숙제, 평가→시험) |
| CTA 버튼 통일 | "+ 새로 만들기"로 일관된 라벨 사용 |
| 카드 정보 가독성 | 제목 2줄 허용, 상대시간 표시, 프로그레스 바 |

### Priority 2 - 단기 개선

| 항목 | 설명 |
|------|------|
| Empty State 개선 | 빈 상태에서 액션 유도 추가 |
| 필터/검색 UX | 접을 수 있는 사이드바, 필터 칩 표시 |
| 교과서 뷰어 도구 | 툴팁 라벨, 확인하기 버튼 강조 |

### Priority 3 - 중기 개선

| 항목 | 설명 |
|------|------|
| 시각적 구분 강화 | 섹션별 Accent Color 적용 |
| 대시보드 재설계 | 우선순위 기반 레이아웃 |
| 반응형 | 테이블/카드 뷰 옵션 |

### Priority 4 - 장기 개선

| 항목 | 설명 |
|------|------|
| 온보딩 | 신규 사용자 가이드 투어 |
| 마이크로 인터랙션 | 완료/성공 시 축하 애니메이션 |
| 접근성 | WCAG AA 기준 충족 |
