# PRD: 이리사 아이템 – 종합 분석기 (aka "그래서 그랬구나!")

## 1. 제품 개요

### 1.1 아이템 정의
- 본 아이템은 **기존 서비스 통합 플랫폼에 탑재되는 단일 서비스 모듈**이다.
- 목적은 교사가 수업 중·후에 생성되는 다양한 산출물(풀이 과정, SEL 검사, 평가 결과 등)을 **한 번에 분석**하여
  - 교사용 Fact 분석
  - 학생/학부모용 이해 가능한 코멘트
  - 학교생활기록부(생기부) 작성 가이드
  를 자동 생성하는 것이다.

### 1.2 문제 정의 (Problem)
- 교사는 다음을 모두 수행하기 어렵다.
  - 풀이 ‘과정’ 분석 (Think Aloud)
  - 인지·정서 데이터(SEL) 해석
  - 평가 결과와의 종합 연결
  - 이를 상담·생기부 문장으로 재가공
- 결과적으로 **데이터는 있으나 인사이트로 전환되지 못하는 상태**가 발생한다.

### 1.3 해결 방향 (Solution)
- 한정된 입력 데이터로 시작하는 **종합 분석 AI 에이전트** 제공
- “결과가 아닌 과정” + “정서/태도” + “코칭 문장화”에 집중
- MVP에서는 **초등/중등 수학 · 학생 1명**을 기준으로 설계

---

## 2. 목표 및 성공 기준

### 2.1 제품 목표
- 교사 기준:
  - 수업 후 정리·상담·기록 업무 시간을 **30분 → 5분 이내**로 단축
- 플랫폼 기준:
  - 향후 N명·N과목·N데이터 확장이 가능한 **표준 분석 아이템** 확보

### 2.2 MVP 성공 기준
- 교사 테스트 기준
  - 산출물 그대로 상담/생기부에 사용 가능하다고 응답 ≥ 70%
- 기술 기준
  - 프롬프트 수정만으로 산출물 톤·구조 변경 가능

---

## 3. 사용자 정의

### 3.1 핵심 사용자
- 초·중등 교사 (수학)

### 3.2 사용 시나리오 (MVP)
1. 교사가 학급/학생 정보를 입력
2. 풀이 활동 영상 + SEL 검사지를 업로드
3. [분석 생성] 버튼 클릭
4. AI가 종합 분석 결과 생성
5. 교사는 결과를 미리보고 Word/PDF로 다운로드

---

## 4. 기능 범위 (Scope)

### 4.1 포함 (In-Scope)
- 학생 1명 기준 종합 분석
- 입력 데이터:
  - 매쓰캔버스 풀이 활동 미디어 (영상/이미지)
  - 학습심리정서검사 결과지 (2종)
- 출력 산출물:
  - 산출물 1: 교사용 Fact 분석
  - 산출물 2: 학생/학부모용 분석 결과

### 4.2 제외 (Out-of-Scope, MVP)
- 다인(多人) 학생 동시 분석
- 영어 과목
- 평가 결과지 자동 연동
- 학사 시스템 연동

---

## 5. 화면 및 UX 구조

### 5.1 전체 레이아웃
- 좌측: 입력 영역
- 우측: 결과 영역

### 5.2 좌측 – 생성 전 입력

#### (1) 학급 정보
- 학교급: 초/중/고 (MVP: 초/중 노출)
- 학년: 1~6 (중등은 1~3)
- 과목: 수학 (고정)
- 분석 대상 학생 수: 1명 (고정)

#### (2) 자료 정보
- 풀이 활동 미디어 업로드
- SEL 검사 결과지 업로드
- 자료 선택은 **옵션형(체크박스)**
- 파일명 규칙 안내 문구 필수

### 5.3 우측 – 생성 후

#### 산출물 탭 구조
- 탭 1: 선생님용 Fact 분석 결과
- 탭 2: 학생/학부모용 분석 결과

#### 공통 기능
- 다시하기 버튼
- 다운로드 버튼
  - Word / PDF 선택
  - 산출물 1, 2 개별 파일 생성

---

## 6. AI 산출물 설계 (핵심)

### 6.1 산출물 1: 교사용 Fact 분석

#### 입력 1. 매쓰캔버스 풀이 활동
- 분석 관점: 학습적 측면
- 기준:
  - 22개정 교육과정
  - 선행학습 여부 (X 기준)

**출력 구조**
- 정답 시:
  - 칭찬 포인트 + 이유
- 오답 시:
  - 격려 포인트 + 이유

#### Next Step
- 정답:
  - 심화 학습용 유사 문항 3개 (객관식)
- 오답:
  - 강화 학습용 유사 문항 3개 + 힌트

---

### 6.2 산출물 2: 학생/학부모용 분석

#### 입력 2. 학습심리정서검사 (2종)
- 분석 관점: 사회·정서 교육

**출력 구조**
- 강점 포인트 + 이유
- 약점 포인트 + 이유
- 2개 검사 종합 분석

**Next Step**
- 강점 강화 방법
- 약점 보완 방법
- 학생용 상담 멘트
- 학부모용 상담 멘트

**톤 가이드**
- 학생 눈높이
- 각 포인트 10줄 이내

---

### 6.3 학교생활기록 가이드

- 대상: 초/중/고 구분
- 입력 데이터: 6.1 + 6.2 결과 기반

**출력**
- 생기부 작성용 문장
- 학습 코칭 종합 의견 (5줄 이내)
- 심리·정서 코칭 종합 의견 (5줄 이내)

---

## 7. 바이브 코딩 기준 PRD (중요)

### 7.1 개발 원칙
- 코드보다 **프롬프트가 중심**
- UI는 최소, 구조는 명확
- 하드코딩 금지, 전부 설정/프롬프트로 제어

### 7.2 기술 스택 가이드 (예시)
- Front: Next.js / React (단일 페이지)
- Back: 서버리스 API
- AI: LLM API (프롬프트 분리 관리)

### 7.3 프롬프트 구조
- Prompt 1: 학습 과정 분석
- Prompt 2: SEL 종합 분석
- Prompt 3: 코칭/문장화 전용

→ 각 Prompt는 **독립 교체 가능**해야 함

---

## 8. 단계별 개발 로드맵

### Step 1 (학생 1명 · 초등 수학)
- 데모 페이지
- AI 토큰 연결
- 핵심 산출물 생성

### Step 2 (학생 4명 테스트)
- 입력 구조 확장 대비
- 프롬프트 보정

### Step 3 (정식 아이템화)
- 협업 규칙 반영
- 플랫폼 탑재
