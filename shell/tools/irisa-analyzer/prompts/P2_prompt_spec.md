
# Prompt Spec – P2 (학습심리·정서 분석)
## P1 연계 및 교사 화면 분리 원칙 포함

본 문서는 **P2 프롬프트 설계 규격서**이며,
P1(학습 과정 분석)과 명확히 구분되는
**학습심리·정서 데이터 기반 산출물 생성**을 목적으로 한다.

본 문서는 P1 프롬프트와 **동시에 운영되는 것을 전제**로 설계되었다.

---

## 0. P2 설계 목적과 범위

### 0.1 목적

P2는 다음 질문에 답하기 위한 프롬프트이다.

> “이 학생은 보통 어떤 방식으로 공부하고,  
> 어떤 심리·정서적 자원을 가지고 있으며,  
> 어떤 지점을 디딤돌로 삼을 수 있는가?”

---

### 0.2 범위 정의 (P1과의 구분)

| 구분 | P1 | P2 |
|---|---|---|
| 초점 | 학습 과정 | 학습 심리·정서 |
| 데이터 성격 | 단기·상황적 | 중·장기·누적 |
| 입력 | 풀이·관찰·평가 과정 | 검사·리포트 결과 |
| 산출물 성격 | 즉각적 연습·다음 행동 | 디딤돌·주의 포인트 코칭 |
| 시간 축 | 이번 수업/단원 | 학기/연간 |

👉 P2는 **P1의 결과를 재해석하지 않는다.**

---

## 1. P2 기본 설계 원칙

1. P2는 **학생 1명(Student Atomic Unit)**만 분석한다.
2. 기존 검사 결과를 **재채점·재판정하지 않는다.**
3. 점수 자체가 아닌 **상대적 강·약 포인트**를 중심으로 정리한다.
4. 진단·치료·낙인 표현은 사용하지 않는다.
5. 기준표가 없는 경우, 설명·요약 중심으로 산출한다.

---

## 2. P2 입력 데이터 범위 (확장 가능)

### 2.1 현재 MVP 입력

- 자기조절학습검사 (학생용 / 교사용)
- 학습종합검사 (학생용 / 교사용)

👉 특정 검사명에 종속되지 않으며,
**영역 + 항목 + 점수(또는 수준)** 구조를 가진 데이터라면 확장 가능하다.

---

### 2.2 입력 데이터 개념 구조

```text
[검사 메타 정보]
- 검사명
- 검사 목적

[영역별 결과]
- 영역명
  - 항목명
  - 점수 또는 수준
```

---

## 3. 핵심 산출 개념 정의

### 3.1 강점 포인트 (Strength Point)

- 학생 결과 중 **유독 높게 나타난 항목**
- 학습·정서적 자원으로 활용 가능한 요소

---

### 3.2 주의 포인트 (Attention Point)

- 학생 결과 중 **유독 낮게 나타난 항목**
- 즉각적 개입이 아닌, 관찰과 보완이 필요한 지점

⚠️ 문제·결함·위험 표현 금지

---

## 4. P2.A – 현재 상태 요약 (Fact-based)

### 역할

- 검사 결과에 근거한 **강점/주의 포인트 구조화**
- 해석·처방·코칭 없음

### 출력 포맷

```text
[강점으로 나타난 포인트]
- 영역/항목 기반 요약

[관찰이 필요한 포인트]
- 영역/항목 기반 요약
```

👉 교사용 내부 참고용

---

## 5. P2.B – Next Step 코칭 산출

### 역할

- 강점·주의 포인트를 **행동 단위 코칭**으로 변환
- 학생·학부모·교사 모두 이해 가능한 표현

### 출력 포맷

```text
[이 학생의 학습·정서 강점 포인트]

[조금 더 살펴보면 좋을 포인트]

[다음으로 시도해 볼 수 있는 행동]
```

---

## 6. 점수 구간별 코칭 기준표 연동 (추후 업데이트)

⚠️ 현재 단계에서는 기준표가 제공되지 않음

### 향후 연동 예정 데이터

- 항목별 점수 구간 정의
- 구간별 권장 코칭 방향
- 표현 톤 가이드

### 연동 원칙

- 기준표 존재 시 → 행동 제안 정밀화
- 기준표 부재 시 → 요약·설명 중심 유지
- 프롬프트 구조 자체는 변경하지 않음

---

## 7. 교사 화면에서의 P1 / P2 분리 원칙

### 7.1 정보 역할 분리

- P1 화면:
  - 이번 수업·평가 기준
  - 풀이 과정, 즉각적 연습

- P2 화면:
  - 학생의 전반적 학습 태도·정서
  - 디딤돌/주의 포인트

---

### 7.2 UI/UX 분리 가이드 (개념)

```text
학생 분석
 ├─ P1. 학습 과정
 │   ├─ 이번 단원 분석
 │   └─ 연습 문제
 └─ P2. 학습 심리·정서
     ├─ 학습 디딤돌
     └─ 행동 코칭
```

👉 두 정보는 **같이 보되, 섞이지 않게** 설계한다.

---

## 8. 금지 영역

P2 프롬프트는 다음을 수행하지 않는다.

- 점수 기반 재평가
- 학생 간 비교
- 진단·치료·위험군 분류
- 사분면(Map) 해석

---

## 9. 요약

- P2는 “판단 AI”가 아니다.
- **검사 결과를 행동 언어로 번역하는 AI**다.
- P1과 병행 사용되도록 설계한다.
- 기준표는 확장 요소로 연결한다.

본 문서는 P2 프롬프트 구현의 **1차 기준 문서**로 사용한다.
