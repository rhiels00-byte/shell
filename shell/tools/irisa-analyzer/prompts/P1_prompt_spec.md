
# Prompt Spec – P1 (풀이·학습 과정)

본 문서는 **Input 1: 학생 풀이·평가·관찰 데이터**에 대한  
프롬프트 설계 **최종 통합 명세서**이다.

P1은 **학생 1명(Student Atomic Unit)**을 최소 단위로 하며,  
학생 수가 N명인 경우 시스템이 학생별로 반복 호출한다.

---

## 0. P1 전체 설계 원칙 (공통)

1. 프롬프트는 항상 **학생 1명만** 분석한다.
2. 학생 간 비교, 집계, 서열화는 수행하지 않는다.
3. 파일 해석·학생 구분·데이터 정합성 판단은 수행하지 않는다.
4. 기준 문서(Standard Reference)는 **참조 기준**으로만 사용한다.
5. 출력은 본 문서에 정의된 포맷만 생성한다.

---

## 1. 프롬프트 네이밍 규칙

### 1.1 기본 규칙

```
P{Input번호}.{단계코드} – {역할 설명}
```

- P1: 풀이·평가·관찰 기반 분석
- 단계코드
  - A: 현재 분석 (Analysis / Fact)
  - B: Next Step 도출 (Recommendation)

A 단계와 B 단계는 **절대 혼합하지 않는다.**

---

## 2. P1 입력 구조 (공통)

```text
[학생 메타 정보]
- student_id
- 학생 식별자(이름 또는 코드)
- 학교급 / 학년

[학생 데이터 요약]
- 학습 수행 데이터 요약
  (문제 풀이, 평가 결과, 수행평가, 교사 관찰 기록 등)
- 데이터 출처 요약
  (파일 종류, 문제 수, 관찰 횟수 등)

[적용 기준 문서]
- 기준 문서 목록 (없을 수도 있음)
- 문서 목적
  - 해석 기준
  - 서술 기준
  - 정책/제약 조건
```

※ 모든 입력은 **이미 학생 1명 기준으로 정규화**된 상태임이 보장된다.

---

# 3. P1.A – 현재 분석 (Fact Analysis)

## 3.1 역할 정의

P1.A는 학생의 학습 데이터를 기반으로  
**관찰 가능한 사실(Fact)**만을 분석·정리한다.

- 원인 추정 ❌
- 처방·조언 ❌
- 정서적 평가 ❌

👉 교사용 **내부 분석 산출물**이다.

---

## 3.2 분석 범위

- 문제 풀이 과정의 행동 특징
- 평가 수행 양상 및 반복 패턴
- 오류 유형의 관찰적 분류
- 교사 관찰 기록에서 드러난 행동

수행하지 않는 것:
- 학생 간 비교
- 평균 대비 판단
- 점수 기반 등급화

---

## 3.3 기준 문서 반영 방식

- 분석 대상은 바꾸지 않는다.
- 관찰 포인트와 서술 관점만 조정한다.

---

## 3.4 출력 포맷 (고정)

```text
[관찰된 학습·행동 특징]
- 객관적으로 관찰된 사실

[강점으로 나타난 지점]
- 반복적으로 긍정적으로 관찰된 요소

[추가 관찰이 필요한 지점]
- 불안정하거나 일관되지 않은 요소
```

---

# 4. P1.B – Next Step 도출  
### (Audience-aware Recommendation)

## 4.1 역할 정의

P1.B는 P1.A 결과를 바탕으로  
**학생의 다음 학습·행동 방향**을 제안한다.

---

## 4.2 대상 분기

```text
audience_type
- TEACHER
- STUDENT_PARENT
```

---

## 4.3 출력 포맷 (고정)

```text
[현재 상태 요약]
- 학생의 현재 학습·행동 상태

[다음에 시도해 볼 방향]
- 2~3개의 구체적인 행동 제안

[연습 또는 적용 예시]
- 동일 맥락의 간단한 예시 또는 활동
```

---

## 5. 학생 수에 따른 사용 규칙

- 학생 1명:
  - 단일 문서 출력 가능
- 학생 N명:
  - 학생별 개별 출력
  - 비교·종합·사분면 해석 수행 ❌

---

## 6. 내부 공유용 정의

- **P1.A**  
  “학생 데이터를 교사의 관찰 기록으로 정리하는 프롬프트”

- **P1.B**  
  “관찰 결과를 학생 눈높이의 다음 행동으로 번역하는 프롬프트”
